Посмотреть решение задачи

Создайте страницу с разметкой, в которой будет отображаться Ваша контактная информация. Используя микроданные и словарь https://schema.org/Person добавьте дополнительные данные в HTML разметку. Проверьте правильность оформления документа с помощью инструмента проверки структурированных данных
https://search.google.com/structured-data/testing-tool
или
http://www.google.com/webmasters/tools/richsnippets
Страница должна быть разделена на две части:
левая часть - список товаров,
правая часть - корзина.
Реализуйте добавление товаров в корзину с помощью drag & drop.

Этот пример хорошо подходит для изучения возможности удаления элемента из списка используя механизм Drag&Drop. Здесь применяется порядок подписки на события, который проясняет алгоритм удаления элемента из списка через Drag&Drop.

  • Микроданные обогащают DOM парами «имя - значение», взятыми из пользовательских словарей и имеющими определенную область видимости.
  • Центральное понятие в системе микроданных - пользовательский словарь.
  • Микроданные - это способ присвоить данным, которые присутствуют на странице и в пользовательском словаре, дополнительную семантику.
  • Создание пользовательского словаря: достаточно выбрать URL на котролируемом Вами домене, чтобы создать вполне уникальный идентификатор.
  • itemtype - атрибут, который обозначает, какой именно словарь микроданных исползуется.
  • itemscope - атрибут, который обозначает область видимоости словаря.
  • itemprop - который устанавливает значение свойств «слова» словаря микроданных.
  • Для браузеров в спецификации HTML5 определен набор функций DOM API, которые позволяют извлекать из web-страниц целостные ообъекты микроданных, свойства и значения свойств.
  • Google поддерживает микроданные в рамках программы Rich Snippets. Когда программа-краулер Google разбирает страницу и находит при этом свойства микроданных, имена которых удовлетворяют словарю, значения таких свойств извлекаются и соххраняются.
  • DOM Inspector - анализатор web-страниц, который выдает много информации о каждом элементе в HTML-странице и о связи каждого элемента с другими. Плюс, к этому, выбрав, например, любой div, можно увидеть его рамку на самой странице. Это очень удобно при сложной структуре HTML-элементов, в которых нужно разобраться.
  • Сделать элемент перетаскиваемым очень просто. Достаточно обавить атрибут draggable=true. Перетаскивать можно что угодно (любые узлы DOM): рисунки, ссылки и т.п.
  • События в разметке Drag-Drop
    • dragstart - вызывается в самом начале переноса «перетаскиваемого» элемента.
    • dragenв - вызывается в конце события перетаскиваня - как успешного, так и отмененного.
    • dragenter - вызывается на зоне приема, когда перетаскиваемый объект впервые оказывается над ней.
    • dragleave - вызывается, когда перетаскиваемый объект покидает зону приема.
    • dragover - непрерывно вызывается, когда перетаскиваемый объект находится над зоной приема.
    • drop - вызывается, когда событие перетаскивания завершается отпусканием элемента над зоной приема.
  • Основополагающим для API геолокации считается новое свойство navigator.geolocation глобального объекта navigatir.
  • Свойство объекта position: coords.longitude, coords.accuracy, coords.altitude, coords.heading, coords.speed, timestamp.
  • handle_error - обработка ошибок. Это аргумент функции getCurrentPosition(). Возможные значения:
    • PERMISSION_DENIED (1) - если пользователь нажмет кнопку Don.t Share (Не сообщать) на панели геолокации или иным способом запретит доступ к данным о своем местонахождении.
    • POSITION_UNAVAILABLE (2) - если не работает сеть и невозможно связаться со спутником позиционирования.
    • TIMEOUT (3) - когда сеть работает, но на расчет координат уходит слишком много времени.
    • UNKNOWN_ERROR (0) - ели прооблема в чем-то другом.
  • watchPosition - функция обратного вызова, будет выполняться каждый раз, когда местонахождение пользователя измениться.
  • updateLocation - метод функции getCurrentPosition(), который запуститься после получения данных о местонахождении.
  • У функции getCurrentPosition() есть необязательный третий аргумент - объект PositionOptions. В нем можно устанавливать значения трех свойств: enableHighAccuracy, timeout long, maximumAge.
